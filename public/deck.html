The Challenge

“Real-time” wealth data is usually anything but.

Advisors and clients expect portfolio values that move with the market. But the data behind them is split across systems that were never designed to work together in real time.

🕘
Stale by design
Positions live in an OLTP store; analytics run on a warehouse refreshed by overnight batch. Dashboards show yesterday.
🧩
Too many moving parts
OLTP + ETL/CDC + warehouse + cache = four systems to license, integrate, monitor, and reconcile.
💸
Costly & brittle
Every copy of the data adds latency, spend, and a new place for numbers to disagree.

The Live Revaluation

Firm AUM is a query, not a cached number.

Every poll re-joins ~23,000 positions to the streaming price table. Nothing is precomputed — the “real-time” value is computed on read, on the smallest instance, in under half a second.

-- recomputed on every dashboard poll · no precompute, no cache SELECT SUM(p.quantity * lp.price) AS aum, SUM(p.quantity * (lp.price - lp.prev_close)) AS day_pnl FROM positions p JOIN latest_price lp ON p.symbol = lp.symbol;
